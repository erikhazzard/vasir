# Adaptive Mastery and Scaffolding

Use this when designing challenge, progression, AI coaching, difficulty, tutorials, dungeon learning, PvP training, or retention recovery.

## Core prior rewrite

Bad default: difficulty is a scalar. If players fail, lower numbers; if they succeed, raise numbers.

Replacement: diagnose the kind of stuckness. Preserve productive struggle, interrupt wheel-spinning, and route players toward the next useful challenge, teacher, cohort, or role.

## Stuckness taxonomy

| State | Evidence | Response |
|---|---|---|
| under-challenged | high success, low arousal, low learning | add mastery goals, route upward, vary constraints |
| productively struggling | retries improve, morale stable | keep challenge, add precise feedback |
| wheel-spinning | repeated attempts, no concept gain | change representation, reveal missing concept, offer mentor |
| socially blocked | competence ok, group unavailable or hostile | route to compatible cohort, AI bridge, learning tag |
| identity blocked | role fantasy mismatches task | alternate contribution path |
| time blocked | wants progress, lacks session length | partial-credit objectives, resumable contracts |
| overwhelmed | too many systems at once | reduce simultaneous variables, preserve autonomy |

## AI-native scaffolding

AI should not simply generate easier content. It should infer:

```text
what the player is trying to become
what concept is missing
what social context would help
what next challenge preserves dignity
what should remain hard
```

## Design knobs

- challenge bands;
- learning-mode tags;
- mechanic-specific coaching;
- mentor queues;
- ghost/replay examples;
- partial objective credit;
- alternate roles;
- adaptive explanations;
- post-failure “next attempt” plans;
- skill cards tied to real group obligations.

## Anti-patterns

- **Pity scaling**: secretly lowering difficulty until victory feels fake.
- **Guide outsourcing**: requiring YouTube/Wikis for basic competence.
- **Failure theater**: repeated wipes with no clearer model.
- **AI overhelp**: the assistant completes the challenge and steals competence.
- **Timer cruelty**: time pressure before shared mental models exist.
