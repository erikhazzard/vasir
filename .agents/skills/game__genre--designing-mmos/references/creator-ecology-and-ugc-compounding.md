# Creator Ecology and UGC Compounding

Use this when designing UGC, AI co-creation, player-authored quests, venues, dungeons, cosmetics, maps, lore, events, or creator economies.

## Core prior rewrite

Bad default: AI-native UGC means infinite content generation.

Replacement: AI-native UGC is a governed creator ecology. Players create, AI drafts, NPCs contextualize, systems validate, communities curate, the world canonizes selectively, and operators govern the substrate.

## Creator pipeline

```text
spark → draft → validate → stage → playtest → publish → route → remix → canonize/archive → reward/steward
```

## Roles

| Actor | Role |
|---|---|
| player creator | taste, intent, social context, original spark |
| AI assistant | drafting, variation, translation, balancing suggestions |
| NPC/world agent | in-world framing and continuity |
| system validator | legality, resource conservation, reward bounds, collision check |
| community | curation, norms, remix, reputation |
| operator | substrate law, high-level canon, abuse/exploit control |
| archive | memory, provenance, lineage |

## Design rules

1. Infinite content needs curation.
2. Infinite authorship needs canon policy.
3. AI-generated rewards need conservation.
4. Remix needs provenance.
5. Discovery must route to creators without winner-take-all collapse.
6. Creator work must compound into identity, reputation, and institutions.
7. AI should lower authoring friction without erasing human taste.

## Canon tiers

| Tier | Meaning |
|---|---|
| private | personal/local texture |
| group | squad/guild/venue history |
| district | visible to local world region |
| seasonal | event-bounded public layer |
| world canon | operator or governance-ratified public history |
| archive | preserved but not active |
