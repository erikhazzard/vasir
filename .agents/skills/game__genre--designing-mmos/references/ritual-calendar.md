# Ritual Calendar

Use this when the design problem involves cadence, seasons, anniversaries, world events, returnability, initiation, celebration, or public belonging.

## Core Principle

A massive world needs a **civic calendar**, not just a reward calendar.

Rituals build commitment when they create shared public attention, emotional intensity, and remembered recurrence.

## Ritual types

| Type | Function |
|---|---|
| Initiation ritual | Turns newcomers into members. |
| Returner ritual | Makes re-entry feel welcomed and legible. |
| Commemorative ritual | Preserves shared memory of loss, victory, migration, or disaster. |
| Campaign ritual | Opens or closes a season, war, or political cycle. |
| Market / craft ritual | Creates recurring economic congregation. |
| Venue / cultural ritual | Creates noncombat cultural identity. |
| Civic festival | Makes the world feel like a society rather than a content queue. |

## Design rules

### 1. Ritual is not the same as repetition

A daily checklist is not a ritual. A ritual has:

- a recognized occasion,
- a shared focus of attention,
- visible participation,
- emotional or symbolic meaning,
- memory afterward.

### 2. Ritual should create belonging, not just obligation

If the ritual is only there to protect retention curves, players feel it.

### 3. Tie rituals to places and institutions

Good rituals strengthen:

- districts,
- guilds,
- coalitions,
- professions,
- cultures,
- world history.

### 4. Use returner rituals deliberately

Returnability improves when coming back feels socially legible: recap weeks, mentor welcomes, anniversary paths, restored titles, returner queues, reactivation banners.

## Failure modes

- chores mistaken for ritual;
- seasons that erase identity;
- festivals with no place memory;
- anniversaries with no connection to actual world history;
- returners treated as lapsed labor rather than valued citizens.

## Metrics

- ritual attendance and repeat attendance;
- emotional recall in surveys or community language;
- post-ritual institution joins or reactivations;
- returner conversion after ritual windows;
- place-based repeat attendance;
- screenshot/chronicle creation around rituals.
