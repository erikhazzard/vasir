# Apprenticeship and Mastery Ladders

Use this when designing onboarding, professions, raids, PvP leadership, crafting, RP venues, creator tools, or AI-human learning systems.

## Core prior rewrite

Bad default: onboarding means tutorials, explanations, and simplified solo tasks.

Replacement: onboarding should create legitimate peripheral participation. New players become useful at the edge of real practice, then move inward as competence and identity grow.

## Mastery ladder

```text
observe → shadow → assist → own a small responsibility → perform under support → teach a narrower task → steward a domain
```

## Peripheral roles by domain

| Domain | Legitimate peripheral roles |
|---|---|
| dungeon/raid | marker assistant, mechanic spotter, backup interrupt, consumable checker, post-wipe scribe |
| PvP/war | scout, supply runner, rear guard, rally pinger, casualty spotter |
| economy | courier, price scout, apprentice crafter, materials sorter, quality inspector |
| guild | greeter, roster helper, event assistant, calendar scribe, mentor shadow |
| RP/venue | door host, photographer, lore aide, scene assistant, guest guide |
| exploration | cartographer apprentice, landmark reporter, route tester |
| AI-native quests | witness, investigator, contract clerk, local informant, validation assistant |

## Design rules

1. A peripheral role must matter to the real activity.
2. It must carry low blame.
3. It must be visible enough to earn recognition.
4. It must teach a transferable concept.
5. It must have a next step.
6. It must not become permanent underclass labor.

## Failure modes

- Tutorial island produces competence with UI but no social identity.
- “Newbie run” removes all responsibility, so newcomers remain spectators.
- Peripheral work becomes unpaid chore labor.
- Expert spaces have no safe edges.
- AI assistants solve tasks instead of helping players become useful.

## Design questions

- What is the first useful thing a new player can do for another player?
- What responsibility can be handed off safely?
- How does the system distinguish “learning role” from “bad performance”?
- What public proof marks the transition from apprentice to practitioner?
- What does the world do when the apprentice becomes a teacher?
