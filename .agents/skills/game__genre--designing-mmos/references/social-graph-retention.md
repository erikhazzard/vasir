# Social Graph Retention

Use this when analyzing churn, matchmaking, cohorting, friend systems, guild health, or why efficient content still fails to retain.

## Core prior rewrite

Bad default: retention is an individual content or reward problem.

Replacement: retention is also graph position. A player returns when they are embedded, needed, expected, recognized, and bridged into future opportunities.

## Social graph states

| State | Risk | Intervention |
|---|---|---|
| isolated active | content may retain temporarily, social churn risk high | route to low-pressure repeated weak ties |
| one-bond dependent | one friend leaving may cascade | add common identity and backup cohorts |
| clique-closed | strong trust, low discovery | add brokerage events and intergroup projects |
| broker | bridges groups, high cultural value | protect from burnout, recognize connective labor |
| hidden glue | organizes but is not formally leader | surface contribution and succession paths |
| leader bottleneck | group depends on one person | delegate, create officer roles, automate admin burden |
| dormant-but-valued | absent but still identity-rich | reactivation rites, patch briefings, social invitations |

## Churn as missing-person design

Ask not only “will this player come back?” but:

```text
Who would notice if this player disappeared?
Who would invite them back?
What role would they return to?
What changed while they were away?
What social proof tells them they still belong?
```

## Metrics

| Metric | Use | Counter-metric |
|---|---|---|
| repeat grouping | recognition | same-clique saturation |
| new durable ties | retention health | spam friend adds |
| bridge count | brokerage | broker burnout |
| group density | trust | clique isolation |
| missed-player pings | returnability | coercive nagging |
| guild subgroup count | integration | factionalism |
| cohort survival | onboarding | inactive zombie groups |

## Design surfaces

- recent useful allies;
- “played well together” prompts;
- apprenticeship routing;
- return invitations from real cohorts;
- leader/facilitator recognition;
- graph-aware event suggestions;
- weak-tie repeat surfaces;
- bridge rewards for connecting groups;
- dormancy and alumni states.

## AI-native implication

AI should route players toward future relationship value, not just content completion. A good social router asks:

```text
Which pairing creates competence, trust, apprenticeship, brokerage, or belonging without exploiting either player?
```
