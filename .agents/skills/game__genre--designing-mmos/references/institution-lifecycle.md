# Institution Lifecycle

Use this when a system touches guilds, squads, statics, coalitions, venue crews, councils, markets, or any long-lived player group.

## Core Principle

Institutions do not just form. They **age**.

A design that supports recruitment but not succession, subgroup drift, dormancy, merger, and reactivation is incomplete.

## Lifecycle stages

| Stage | Primary need | Typical risk |
|---|---|---|
| Founding | Discovery, charter, initial cohesion, early wins. | Empty shell, spam recruitment, no shared purpose. |
| Growth | Onboarding, subgroup formation, role coverage. | New members remain spectators. |
| Maturity | Delegation, calendars, policy clarity, cross-group cohesion. | Officer overload, clique hardening. |
| Strain | Succession, conflict handling, mission drift, uneven participation. | One subgroup leaves and takes the culture with it. |
| Dormancy | Alumni memory, low-effort re-entry, archive, stewardship. | Silent death and lost identity. |
| Reactivation / merger | Cultural translation, roster reconciliation, role remapping, shared wins. | Culture clash, loss of trust, symbolic capture. |

## Five stress layers

Review institutional failure on five layers:

1. personal,
2. subgroup,
3. institution,
4. game/system,
5. operator/company.

Do not attribute every institutional failure to “bad leaders.” Sometimes the system created the brittleness.

## Design rules

### 1. Leadership must be delegable

If one person owns the entire institution surface, burnout and fragility are guaranteed.

### 2. Onboarding must lead to play, not mere membership

Measure whether newcomers actually play with existing members inside a short time window.

### 3. Subgroups should overlap

Purely isolated subgroups make later fracture more likely. Institutions need overlap events, cross-team work, and multi-role visibility.

### 4. Dormancy is not failure if memory survives

A dormant guild, venue, or crew should be able to preserve:

- charter,
- roster memory,
- history,
- honors,
- key places,
- easy reactivation.

### 5. Mergers need cultural translation

Institution growth is not only individual onboarding. Sometimes groups merge. Mergers need:

- visible mission match,
- title/role mapping,
- policy reconciliation,
- archive integration,
- symbolic fairness.

## Useful tools

- charters and policy surfaces;
- squad / team substructures;
- officer delegation;
- event history;
- alumni status;
- dormancy mode;
- merger wizard / roster mapping;
- succession handoff;
- conflict notes and governance logs;
- reactivation campaigns.

## Metrics

- newcomer time-to-first-play;
- subgroup overlap;
- leader concentration index;
- role vacancy duration;
- event cancellation rate;
- officer churn;
- dormant group reactivation rate;
- merger retention after 30/90 days.
