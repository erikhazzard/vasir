# Algorithmic Culture and Social Routing

Use this when designing matchmaking, recommendations, feeds, quest surfacing, party finders, AI DMs, event surfacing, friend suggestions, or personalized world routing.

## Core prior rewrite

Bad default: ranking optimizes predicted completion, click, session length, or wait time.

Replacement: in an AI-native MMO, recommendation policy is culture policy. The algorithm decides who meets whom, which places feel alive, which institutions grow, what identities are reinforced, and whether players discover or collapse into bubbles.

## Objective stack

| Objective | Creates | Risk |
|---|---|---|
| skill parity | fair challenge | sterile sameness |
| complementary expertise | apprenticeship | exploitation of teachers |
| repeated weak ties | recognition | stalking-like overexposure |
| common bond | trust | clique closure |
| common identity | civic belonging | factional hardening |
| serendipity | discovery | incoherence |
| local geography | place memory | isolation if overlocal |
| institution pipeline | guild/squad growth | funnel capture |
| diversity | cultural breadth | low compatibility if naïve |
| returner routing | clean reentry | shame if framed poorly |

## Design questions

1. What does the ranking function make culturally common?
2. Who gets visibility and who disappears?
3. Does matching create future relationship value?
4. Does it create apprenticeship opportunities?
5. Does it preserve place memory?
6. Does it trap players in preference bubbles?
7. Does it overuse high-social-value players as unpaid glue?
8. Can players understand and steer why something is recommended?

## Required counter-metrics

- relationship survival after recommendation;
- repeat contact quality;
- serendipity acceptance;
- new institution formation;
- broker burnout;
- creator/venue exposure concentration;
- local place-memory formation;
- filter-bubble depth;
- player-perceived agency.
