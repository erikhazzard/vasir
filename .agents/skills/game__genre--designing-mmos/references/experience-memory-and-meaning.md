# Experience Memory and Meaning

Use this when designing sessions, dungeons, raids, events, quests, endings, rituals, logout flows, recaps, and return hooks.

## Core prior rewrite

Bad default: optimize average fun per minute.

Replacement: players remember peaks, recoveries, endings, social meaning, and unresolved aspiration. A session’s aftertaste can matter more than its average moment.

## Session memory arc

```text
entry promise → rising investment → peak → failure/repair or victory → recognition → ending → next-return hook
```

## Design surfaces

| Moment | Design job |
|---|---|
| entry | state purpose, stakes, who is here, expected conduct |
| peak | create intensity worth retelling |
| failure | make repair and next attempt obvious |
| victory | assign gratitude and recognition, not just loot |
| ending | close socially, summarize meaning, route next step |
| logout/return | leave a clean hook without hostage pressure |

## Meaning sources

- sacrifice;
- rescue;
- risk survived;
- someone noticed;
- public record;
- identity revealed;
- group changed;
- place changed;
- old promise fulfilled;
- new obligation chosen;
- return made easy.

## Design questions

1. What will players retell?
2. What is the emotional peak?
3. What is the ending state?
4. Who gets thanked or recognized?
5. Does failure end with repair or shame?
6. What public/private memory survives?
7. What makes the next session feel invited, not demanded?

## Anti-patterns

- Reward popups replacing social closure.
- Queue instantly dissolves before gratitude.
- Wipe screen becomes prosecution.
- Event ends with no archive.
- Daily reset deletes meaning.
- Login hook is pure FOMO.
