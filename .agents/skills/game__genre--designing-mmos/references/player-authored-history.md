# Player-Authored History

Use this when designing persistent consequences, monuments, records, local legends, battle aftermath, contract archives, political memory, or world storytelling.

## Core Principle

A world becomes historically meaningful when players can **leave traces that outlive the reward payout**.

The goal is not infinite clutter. The goal is selective persistence with provenance.

## The promotion ladder of memory

| Level | What survives | Examples |
|---|---|---|
| Ephemeral | Short-lived local trace. | temporary marker, rumor board, district notice, recent battle scars |
| Local memory | Repeatable place-specific memory. | named camp, district story, convoy route, recurring memorial sign |
| Institutional archive | Group-specific preserved history. | guild chronicle, treaty log, leader record, venue guestbook, contract ledger |
| Public historical record | Region or world-visible preserved event. | battle report, captured territory timeline, monument, seasonal chronicle |
| Canonical world memory | Rare world-defining preserved change. | world monument, permanent named event site, constitutional change |

Not every event deserves canon. Most do not.

## Design rules

### 1. Preserve provenance

Every durable trace should answer:

- who authored it,
- who validated it,
- what changed,
- when it happened,
- who can see it,
- how long it lasts.

### 2. Prefer scaffolds over fully authored chronicles

Players do not need the game to narrate every event. They need structures that let them:

- name events,
- attach participants,
- preserve artifacts,
- revisit places,
- compare versions of history.

### 3. Tie memory to geography when possible

History is stronger when it has a location:

- battleground,
- market square,
- district,
- frontier,
- guild hall,
- memorial wall,
- captured bridge.

### 4. Curate canon deliberately

Player-authored history becomes noise if every action is permanent. Promote selectively using thresholds such as:

- scale,
- rarity,
- institutional relevance,
- recurrence,
- player attention,
- official recognition.

## Systems that help

- contract and treaty archives;
- guild / coalition chronicles;
- district rumor boards;
- map layers showing past events;
- memorial objects and inscriptions;
- provenance on trophies and relics;
- seasonal or campaign almanacs;
- “what happened here?” interactables;
- returner recap surfaces.

## Failure modes

- all history is operator-authored only;
- player action vanishes after reward collection;
- persistence causes unreadable clutter;
- no provenance, so records are meaningless;
- private Discord becomes the only archive.

## Metrics

- archive and chronicle usage;
- revisit rate to memorialized places;
- player naming of locations in chat;
- historical artifact inspection;
- returner use of recap/history surfaces;
- proportion of major world events with preserved traces.
