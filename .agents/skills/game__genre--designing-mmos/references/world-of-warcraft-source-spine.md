# World of Warcraft Source Spine

Use this when making historical or factual claims about WoW’s scale, social research, access tools, cross-realm systems, token economy, Mythic+, Cataclysm world rewrite, or decline. Separate source fact from design interpretation.

## Design Use

When making a WoW-based argument, write it in this structure:

```text
Source fact:
What it supports:
What it does not prove:
Design interpretation:
Modern translation:
```

## High-Confidence WoW Decline Claim

Use this wording by default:

```text
WoW did not simply die, and no single feature killed it. The stronger design diagnosis is that WoW repeatedly optimized access, speed, seasonal execution, and content reach in ways that solved real problems but often removed social residue faster than replacement memory systems appeared.
```

## Claims to Avoid

- “Dungeon Finder killed WoW.”
- “LFR killed raiding.”
- “Cataclysm killed the world.”
- “Tokens made WoW pay-to-win” without specifying the gold-to-status path.
- “Classic proves friction creates community.”
- “WoW was successful because it had millions of players.”
- “Retail decline proves convenience is bad.”

Replace each with a social-substitution analysis.
